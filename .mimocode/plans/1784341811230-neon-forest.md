# Deploy Frontend + API Server Separately

> **Goal:** Split the Next.js all-in-one app into a frontend (Vercel) and a separate API server (Express on Railway/Render/Fly.io), with Supabase remaining as the hosted backend.

---

## Architecture

```
┌─────────────┐     ┌─────────────────┐     ┌──────────┐
│   Frontend   │────▶│   API Server     │────▶│ Supabase │
│   (Vercel)   │     │  (Express/Rail)  │     │  (Hosted) │
│  Next.js     │     │  Node.js/Express │     │  Postgres │
│  Static+SSR  │     │  /api/* routes   │     │  Auth     │
└─────────────┘     └─────────────────┘     └──────────┘
```

- **Frontend (Vercel):** Next.js static pages + SSR, no API routes
- **API Server (Railway/Render/Fly.io):** Express.js serving all `/api/*` endpoints
- **Supabase:** PostgreSQL, Auth, Storage, Realtime (unchanged)

---

## What Changes

### 1. Create standalone API server

Create `crewwork/api-server/` with:
- `package.json` — Express + dependencies (no Next.js)
- `src/index.ts` — Express app with CORS, routes
- `src/routes/` — one file per API route (setup, provision, invite, upload, storage, ai/chat, ai/compress, livekit/token)
- `.env.example` — all required env vars

### 2. Modify Next.js frontend

- Remove all `src/app/api/` routes
- Create `src/lib/api.ts` — centralized API client that points to the separate server URL
- Update all `fetch('/api/...')` calls to use the new client
- Set `NEXT_PUBLIC_API_URL` env var for the API server URL

### 3. Update frontend fetch calls

Every frontend file that calls `/api/*` needs updating:
- `src/app/setup/page.tsx` — calls `/api/setup`
- `src/components/workspace/invite-dialog.tsx` — calls `/api/setup`, `/api/invite`
- `src/app/auth/page.tsx` — calls `/api/invite/complete`
- `src/app/workspace/layout.tsx` — calls `/api/storage`
- `src/components/profile/profile-edit-dialog.tsx` — calls `/api/upload`
- `src/components/ai/ai-chat.tsx` — calls `/api/ai/chat`
- `src/components/bookmarks/save-post-button.tsx` — calls `/api/ai/compress`
- `src/components/chat/channel-view.tsx` — calls `/api/livekit/token`
- `src/lib/supabase/provisioner.ts` — calls `/api/provision`

## Deployment Steps

### Frontend (Vercel)
1. Connect GitHub repo to Vercel
2. Set root directory to `crewwork/`
3. Set env var `NEXT_PUBLIC_API_URL=https://your-api-server.railway.app`
4. Deploy

### API Server (Railway)
1. Connect GitHub repo to Railway
2. Set root directory to `crewwork/api-server/`
3. Set all env vars (SUPABASE_URL, ANON_KEY, SERVICE_ROLE_KEY, DEEPSEEK_API_KEY, LIVEKIT keys)
4. Deploy

## Verification

1. `cd crewwork/api-server && npm install && npm run dev` — API server starts
2. `cd crewwork && npm run dev` — Frontend starts with `NEXT_PUBLIC_API_URL=http://localhost:3001`
3. Test all flows: setup, auth, channels, calls, AI chat, bookmarks
4. Verify CORS allows frontend domain
