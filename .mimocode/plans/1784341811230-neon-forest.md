# Fix: Channel Settings Button Hidden When Calls Disabled

> **Bug:** The channel settings button (Users icon) is hidden when `channel.calls_enabled === false` because it shares the same conditional wrapper as the Phone button. Users can't access channel settings to re-enable calls once they disable them.

**File:** `src/components/chat/channel-view.tsx` (lines 496-515)

## Current Code (broken)

```tsx
<div className="flex items-center gap-1">
  {!isDirectMessage && channel.calls_enabled !== false && (
    <>
      <button onClick={handleStartCall} ...>Phone</button>
      <button onClick={() => setChannelSettingsOpen(true)} ...>Users</button>
    </>
  )}
</div>
```

Both buttons are inside the `channel.calls_enabled !== false` guard. When calls are disabled, the settings button disappears too.

## Fix

Separate the two buttons so the settings button is always visible for non-DM channels:

```tsx
<div className="flex items-center gap-1">
  {!isDirectMessage && channel.calls_enabled !== false && (
    <button
      onClick={handleStartCall}
      className="h-8 w-8 rounded-lg flex items-center justify-center hover:bg-[#FEF2F2] transition-colors"
      title="Start a call"
    >
      <Phone className="h-4 w-4" style={{ color: '#A8A29E' }} />
    </button>
  )}
  {!isDirectMessage && (
    <button
      onClick={() => setChannelSettingsOpen(true)}
      className="h-8 w-8 rounded-lg flex items-center justify-center hover:bg-[#FEF2F2] transition-colors"
      title="Channel settings & members"
    >
      <Users className="h-4 w-4" style={{ color: '#A8A29E' }} />
    </button>
  )}
</div>
```
