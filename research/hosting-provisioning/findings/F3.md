# F3: Open-source hosting control panels with API capabilities

## Findings

### [1] Virtualmin offers the most comprehensive REST API with 100+ named API programs for full account lifecycle management
- quote: "Virtualmin offers both a command-line API and an HTTP-based remote API for extensive server management, including handling servers, users, databases, etc."
- url: https://www.virtualmin.com/docs/development/remote-api/
- source_type: primary
- published: 2024-01-29 (doc revision)
- confidence: high

### [2] Virtualmin's API includes dedicated create-domain, delete-domain, create-user, create-plan, create-reseller, and create-database programs — covering the full provisioning/deprovisioning cycle
- quote: "Virtual Servers: create-domain, clone-domain, delete-domain, disable-domain, enable-domain, migrate-domain, modify-domain... Account Plans: create-plan, delete-plan, modify-plan, list-plans"
- url: https://www.virtualmin.com/docs/
- source_type: primary
- published: 2024-01-29
- confidence: high

### [3] Virtualmin's remote API is HTTP-based via CGI endpoint, supports JSON/XML output, and authenticates via HTTP basic auth (master admin only)
- quote: "Remote calls must be made through the CGI, e.g: https://yourserver:10000/virtual-server/remote.cgi. Provide a program parameter for the command-line program to invoke."
- url: https://www.virtualmin.com/docs/development/remote-api/
- source_type: primary
- published: 2024-01-29
- confidence: high

### [4] HestiaCP has a REST API enabled by default and a CLI, with API access key authentication supporting granular command-level permissions
- quote: "The Hestia REST API is available to perform core functions of the Control Panel. For example, we use it internally to synchronise DNS clusters and to integrate the WHMCS billing system. The API can also be used to create new user accounts, domains, databases or even to build an alternative web interface."
- url: https://github.com/hestiacp/hestiacp/blob/main/docs/docs/server-administration/rest-api.md
- source_type: primary
- published: 2024 (GitHub)
- confidence: high

### [5] HestiaCP v1.6.0+ introduced access keys with per-command granularity, allowing scoped API tokens for third-party integration (e.g., WHMCS billing)
- quote: "User-specific. Can limit permissions. For example only v-purge-nginx-cache. Ability to disable login via other methods but still allow the use of api keys."
- url: https://github.com/hestiacp/hestiacp/blob/main/docs/docs/server-administration/rest-api.md
- source_type: primary
- published: 2024 (GitHub)
- confidence: high

### [6] HestiaCP supports non-admin API users since v1.6.0, with custom API key profiles defined via files in /usr/local/hestia/data/api/
- quote: "With the release of Hestia v1.6.0, we have introduced a more advanced API system and it will allow non-admin users to use specific commands."
- url: https://github.com/hestiacp/hestiacp/blob/main/docs/docs/server-administration/rest-api.md
- source_type: primary
- published: 2024 (GitHub)
- confidence: high

### [7] HestiaCP is Linux-only (Debian 11/12, Ubuntu 22.04/24.04), requires root installation, and API is activated by default with the `-d` flag
- quote: "Hestia only runs on AMD64 / x86_64 and ARM64 / aarch64 processors... -d, --api Activate API [yes | no] default: yes"
- url: https://docs.hestiacp.com/docs/introduction/getting-started.html
- source_type: primary
- published: unknown (v1.9.6)
- confidence: high

### [8] CyberPanel has an `api/` directory in its source code and is built on OpenLiteSpeed, with 2k GitHub stars and active development (10k+ commits)
- quote: "Cyber Panel - The hosting control panel for OpenLiteSpeed"
- url: https://github.com/usmannasir/CyberPanel
- source_type: primary
- published: 2025 (latest commits)
- confidence: medium

### [9] CyberPanel's official documentation site (cyberpanel.net/docs/) was unreachable during research; local guides exist at github.com/usmannasir/cyberpanel/blob/stable/guides/
- quote: "📚 Complete Guides Index - All available documentation in one place"
- url: https://github.com/usmannasir/CyberPanel/blob/stable/guides/INDEX.md
- source_type: primary
- published: unknown
- confidence: medium

### [10] SolidCP is a Windows-based multi-server control panel (v1.5.1, Dec 2024) supporting Exchange, SharePoint, Hyper-V, and Active Directory — but no public REST API documentation was found
- quote: "SolidCP is optimized for speed and usability, and it's completely free and opensource!"
- url: https://solidcp.com/
- source_type: primary
- published: 2024-12-17 (v1.5.1)
- confidence: medium

### [11] Virtualmin GPL is free and open-source; Virtualmin Professional is paid and adds additional features (the API itself is available in both)
- quote: "Virtualmin offers both a command-line API and an HTTP-based remote API for extensive server management"
- url: https://www.virtualmin.com/docs/
- source_type: primary
- published: unknown
- confidence: high

### [12] HestiaCP has 4.4k GitHub stars and 901 forks, indicating strong community adoption; latest release is v1.9.6
- quote: "hestiacp/hestiacp Public · 4.4k stars · 901 forks"
- url: https://github.com/hestiacp/hestiacp
- source_type: primary
- published: 2025 (latest)
- confidence: high

## Dead ends
- CyberPanel official docs site (cyberpanel.net/docs/*) returned 404 on all URL patterns attempted; documentation appears to be transitioning to a new platform
- SolidCP documentation portal (docs.solidcp.com) DNS resolution failed; development docs at solidcp.com/kb/development/ only cover building from source, not API usage
- CyberPanel GitHub wiki had only 12 static pages with no API documentation
