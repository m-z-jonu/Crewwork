# F5: GitHub repos and npm packages for hosting automation

## Findings

### [1] Shockbyte/whmcs-node is the most popular TypeScript/JS WHMCS API wrapper (13 stars, 62 commits, 14 releases)
- quote: "A Typescript/Javascript API wrapper for WHMCS."
- url: https://github.com/Shockbyte/whmcs-node
- source_type: community
- published: Nov 2023 (latest release v1.0.14)
- confidence: high

### [2] n8n-nodes-whmcs is a fresh (Jun 2026) n8n community node that wraps all 200+ WHMCS API actions into drag-and-drop workflow nodes
- quote: "A community node package that brings the WHMCS billing & automation API into n8n as native nodes... ships two nodes: WHMCS — an action node covering Clients, Orders, Invoices, Products/Services, Tickets, Domains and System operations, plus a Custom API Call escape hatch."
- url: https://www.npmjs.com/package/n8n-nodes-whmcs
- source_type: community
- published: Jun 2026 (v1.0.1, 19 days ago as of Jul 12 2026)

## Dead ends
- GitHub search for "hosting provisioning automation" returned 82 repos but most are Ansible/Terraform cloud provisioning (AWS/IBM Cloud), not cPanel/WHMCS hosting account automation
- No dedicated open-source WHMCS-to-cPanel account provisioning bridge was found (WHMCS has its own cPanel module built-in, so third-party wrappers are unnecessary for that specific flow)
- No open-source Plesk provisioning automation tool of any significance exists on GitHub or npm

## Suggested follow-ups
1. Investigate the WHMCS built-in cPanel/Plesk server modules vs. what third-party modules add — what gaps exist in the native provisioning flow?
2. Look at WHMCS Marketplace addons for hosting provisioning — the proprietary addon ecosystem likely has commercial tools that don't appear on GitHub
3. Examine whether n8n/whmcs-mcp could be used as a workflow orchestration layer to chain WHMCS API calls with cPanel UAPI calls for end-to-end provisioning
