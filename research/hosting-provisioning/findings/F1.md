# F1: Hostinger API capabilities for reseller and account provisioning

## Findings

### [1] Hostinger has an official public REST API currently in beta (v0.12.0) at developers.hostinger.com
- quote: "The Hostinger API provides a comprehensive set of endpoints that allow developers to interact with Hostinger's services programmatically."
- url: https://github.com/hostinger/api
- source_type: primary
- published: 2026-05-25 (last indexed), API active since at least 2025
- confidence: high

### [2] The API covers four domains — VPS, Domains, DNS, and Billing — but does NOT include shared hosting or hosting account management
- quote: "The Hostinger API is organized into four main service domains: Billing (/api/billing/v1/), VPS (/api/vps/v1/), Domains (/api/domains/v1/), DNS (/api/dns/v1/)"
- url: https://deepwiki.com/hostinger/api/2-getting-started
- source_type: primary
- published: 2026-05-25
- confidence: high

### [3] VPS instances can be purchased and provisioned programmatically via POST /api/vps/v1/virtual-machines with item_id, data_center_id, and template_id
- quote: "The POST /api/vps/v1/virtual-machines endpoint handles the purchase. The request body uses item_id (e.g., hostingercom-vps-kvm2-usd-1m) to identify the plan."
- url: https://deepwiki.com/hostinger/api/4.1-virtual-machine-lifecycle
- source_type: primary
- published: 2026-05-25
- confidence: high

### [4] Full VPS lifecycle is API-manageable: create, start/stop/restart, reinstall OS, reset passwords, manage SSH keys, snapshots, firewalls, Docker
- quote: "Server Management: List VPS instances, Start/Stop/Restart/Purchase a VPS, Reinstall operating system, Reset root password, Manage SSH keys and post-install scripts"
- url: https://github.com/hostinger/api-n8n-node
- source_type: primary
- published: 2026 (v1.0.18)
- confidence: high

### [5] The API is single-account scoped — tokens inherit the owning user's permissions with no multi-tenant or reseller model exposed
- quote: "API tokens are created and managed from the Account page in the Hostinger Panel. Each token can optionally be configured with an expiration period for enhanced security."
- url: https://deepwiki.com/hostinger/api/2-getting-started
- source_type: primary
- published: 2026-05-25
- confidence: high

### [6] There is NO reseller API, no sub-account creation endpoint, and no WHMCS-type hosting provisioning module in Hostinger's public API
- quote: (absence of evidence — no endpoint found for creating accounts, managing sub-users, or provisioning shared hosting across any source)
- url: https://developers.hostinger.com
- source_type: primary
- published: 2026-05-25
- confidence: high

### [7] Hostinger provides official SDKs in Python, PHP, TypeScript, plus Ansible collection, Terraform provider, and n8n community node
- quote: "SDKs & Tools: Ansible, PHP, Python, Terraform, TypeScript, n8n Node, MCP Server, Postman Collection"
- url: https://github.com/hostinger/api
- source_type: primary
- published: 2026-05-25
- confidence: high

### [8] The billing catalog API allows listing products/pricing and the deprecated orders endpoint can create orders, but only for VPS and Domains — not shared hosting
- quote: "The general POST /api/billing/v1/orders endpoint is considered deprecated. Integrations should prefer service-specific endpoints which handle both the order and the resource setup in a single flow."
- url: https://deepwiki.com/hostinger/api/3-billing-api
- source_type: primary
- published: 2026-05-25
- confidence: high

### [9] The API requires a Hostinger account and API token obtained from hpanel.hostinger.com/profile/api — no partner/reseller API key tier is documented
- quote: "To get started with the Hostinger API, follow these steps: Sign up for a Hostinger account. Obtain your API token from the Hostinger dashboard."
- url: https://github.com/hostinger/api
- source_type: primary
- published: 2026-05-25
- confidence: high

### [10] Rate limits are enforced with 429 responses; domain availability checks are limited to 10 requests per minute
- quote: "Rate limit headers are included in the response to help you manage your requests. Specific endpoints, such as domain availability checking, have strict limits (e.g., 10 requests per minute)."
- url: https://deepwiki.com/hostinger/api/2-getting-started
- source_type: primary
- published: 2026-05-25
- confidence: high

## Dead ends
- DuckDuckGo HTML search returned a CAPTCHA on the third query — no additional results from that search
- No Hostinger "reseller API" documentation found anywhere — official docs, GitHub, or community sources
- No WHMCS module for Hostinger found in the search results (the WHMCS article was about linking reseller servers generically, not Hostinger-specific)
- No evidence of Hostinger offering a partner/reseller program with API access for account provisioning
